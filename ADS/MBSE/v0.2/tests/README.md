# ADS v0.2 IVV verification plan

> No D2/Capella diagrams were edited. No new external research was performed; this plan uses the saved `tests/references/` corpus and [`../../../../PM&SE/IVV.md`](../../../../PM&SE/IVV.md).

This README is the ADS v0.2 planning entry point. Detailed test-definition diagrams are out of scope here; rows below are a historical v0.2 planning inventory superseded by the all-version definitions under `ADS/MBSE/tests/`; they are not active CON-003 test-definition work.

## 1. SSIV / development-version assumptions

- **Inferred baseline:** `ADS-v0.2` incremental/development SSIV, inferred from `ADS/MBSE/v0.2/` and the model label `ADS v0.1-0.2 Subsystem`.
- **Operational interpretation:** Serial0-to-PC logging is a development verification interface; `5 Hz` is the internal ADS collection rate.
- **Execution records:** exact hardware revision/BOM, firmware commit, GPS truth source, angular-rate truth source, Serial0 schema revision, environmental readings, and trace-label cross-references are report-time records. The active all-version definitions control the pass/fail gates.
- **Traceability assumption:** until upstream traceability nodes exist, activities trace to controlled provisional capabilities/constraints in this README.

## 5. Reporting conventions

Store evidence under `tests/results/<activity-id>/`. Each report shall identify referenced model views/elements, ADS v0.2 hardware/software baseline, as-tested configuration, raw data, analysis scripts, actual conditions, deviations, anomalies, pass/fail rationale, waivers, and retest status. Reports should reference model content rather than duplicate it.

## 6. Coverage gaps and follow-up items

- **Traceability gap:** no explicit mission, capability, use-case, or feared-event model elements exist in the v0.2 D2 views.
- **Modeling gap:** no detailed verification means/test operator/harness are modeled in the functional chains.
- **Fault-hardening gap:** v0.2 lacks an explicit I2C read timeout constraint; v1.0 adds one.
- **Assumption status:** Serial0 frame schema/parser control now lives in `ADS/MBSE/tests/schemas/` and `ADS/MBSE/tests/tools/`; `Measurements [LoRa Frame]` is treated only as a Serial0 development record label, not LoRa delivery evidence.
- **Execution records:** GPS truth source, angular-rate reference, environmental readings, confidence claim, and firmware commit are report-time records.
- **Definition-stage handoff:** all-version ADS definitions under `ADS/MBSE/tests/` supersede this historical inventory; do not edit source ADS diagrams from this version-local planning file.
